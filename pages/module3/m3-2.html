M3.2: Factories met Relaties

Leeruitkomsten

  • Ik kan factories maken voor gerelateerde modellen.
  • Ik kan foreign keys automatisch vullen in een factory.
  • Ik kan datums genereren met Carbon en Faker.

1. Foreign Keys Vullen

Een product heeft een category_id nodig. Om te voorkomen dat we een ID kiezen die niet bestaat, halen we alle categorieën op en kiezen er willekeurig eentje uit.


// database/factories/ProductFactory.php
public function definition(): array
{
    // Zorg dat je de classes importeert!
    return [
        'name' => $this->faker->word,
        'description' => $this->faker->paragraph(15),

        // Pak een willekeurig ID uit de Categories tabel
        'category_id' => Category::all()->random()->id,
    ];
}

2. Datums met Carbon

Voor de prijzen tabel willen we een datum in het verleden. We gebruiken Carbon om makkelijk met datums te rekenen.


// database/factories/PriceFactory.php
use Carbon\Carbon;

public function definition(): array
{
    return [
        'price' => $this->faker->randomFloat(2, 2, 100),

        // Vandaag min een willekeurig aantal dagen (0-365)
        'effdate' => Carbon::today()->subDays(random_int(0, 365)),

        'product_id' => Product::all()->random()->id,
    ];
}
