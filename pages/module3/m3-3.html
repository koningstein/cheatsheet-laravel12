M3.3: Seeders & Magic Methods

Leeruitkomsten

  • Ik kan seeders aanpassen om gerelateerde data aan te maken.
  • Ik begrijp hoe "Magic Methods" in factories werken (bijv. hasPrices).
  • Ik begrijp de volgorde van seeden bij relaties.

1. Magic Methods

In plaats van losse seeders die willekeurige ID's pakken, bouwen we de structuur logisch op. Als je model een relatie prices() heeft, kun je in de factory hasPrices() gebruiken.

2. De ProductSeeder

We maken alles aan vanuit de Categorie. Zo weten we zeker dat elke categorie producten heeft, en elk product prijzen heeft.


// database/seeders/ProductSeeder.php
public function run(): void
{
    // 1. Maak 25 Categorieën
    Category::factory()
        ->times(25)
        // 2. Elke categorie krijgt 5 Producten (relatie: products)
        ->has(
            Product::factory()
                ->times(5)
                // 3. Elk product krijgt 3 Prijzen (relatie: prices)
                ->hasPrices(3)
        )
        ->create();
}

3. DatabaseSeeder Volgorde

Zorg dat de volgorde klopt. Eerst de rollen en users, dan pas de data.


$this->call([
    RoleAndPermissionSeeder::class,
    UserSeeder::class,
    ProductSeeder::class,
]);
