M3.12: Destroy & Database Constraints

Leeruitkomsten

  • Ik kan records veilig verwijderen met foreign key constraints.
  • Ik begrijp het verschil tussen cascade en restrict.
  • Ik kan database exceptions (fouten) opvangen met try-catch.

1. Cascade (Product -> Prices)

In onze migratie (M3.1) hebben we bij prices gezegd: onDelete('cascade'). Als we nu een product verwijderen, ruimt de database automatisch alle prijzen op. Hier hoeven we niets extra's voor te doen in de code!


// ProductController.php
public function destroy(Product $product)
{
    // Dit verwijdert het product EN alle gekoppelde prijzen (door cascade)
    $product->delete();

    return to_route('products.index')
            ->with('status', 'Product verwijderd!');
}

2. Restrict (Category -> Products)

Bij Categorieën hebben we onDelete('restrict') gebruikt. Je mag een categorie NIET verwijderen als er nog producten in zitten. Als je dit toch probeert, geeft de database een harde error.

Om dit netjes af te handelen, gebruiken we een try-catch blok.


// CategoryController.php
public function destroy(Category $category)
{
    try {
        $category->delete();
        return to_route('categories.index')->with('status', 'Categorie verwijderd!');

    } catch (\Throwable $e) {
        // Vang de database fout op en toon een nette melding
        report($e); // Log de technische fout

        return to_route('categories.index')
                ->with('error', 'Kan categorie niet verwijderen: er zijn nog producten gekoppeld!');
    }
}
