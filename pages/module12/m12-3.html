Mail vs Notifications

In Laravel zijn er twee manieren om berichten te versturen: Mailables en Notifications. Hoewel ze op elkaar lijken, is het doel verschillend.

  • Mail: Gebruik dit voor complexe, mooi opgemaakte e-mails (nieuwsbrieven, uitgebreide facturen). Het is specifiek voor e-mail.
  • Notifications: Gebruik dit voor korte status-updates (je bestelling is verzonden, er is op je gereageerd). Het krachtige is dat één notificatie via meerdere kanalen kan gaan: E-mail, SMS, Slack, Database (voor een belletje in je dashboard), etc.

1. Mailables (E-mail only)

Een Mailable is een class die een e-mail representeert. Je maakt deze met php artisan make:mail OrderShipped.

Bestand: app/Mail/OrderShipped.php

use Illuminate\Bus\Queueable;
use Illuminate\Mail\Mailable;
use Illuminate\Mail\Mailables\Content;
use Illuminate\Mail\Mailables\Envelope;

class OrderShipped extends Mailable
{
    use Queueable;

    public function __construct(public Order $order) {}

    public function envelope(): Envelope
    {
        return new Envelope(
            subject: 'Je bestelling #' . $this->order->id . ' is verzonden',
        );
    }

    public function content(): Content
    {
        // Verwijst naar resources/views/emails/orders/shipped.blade.php
        return new Content(view: 'emails.orders.shipped');
    }
}

Versturen gaat via de Mail facade: Mail::to($user)->send(new OrderShipped($order));.

2. Notifications (Multi-channel)

Dit is vaak de betere keuze voor systeemmeldingen. Maak aan met php artisan make:notification InvoicePaid.

Bestand: app/Notifications/InvoicePaid.php

use Illuminate\Notifications\Notification;
use Illuminate\Notifications\Messages\MailMessage;

class InvoicePaid extends Notification
{
    public function __construct(public Invoice $invoice) {}

    // Bepaal via welke kanalen dit verstuurd wordt
    public function via(object $notifiable): array
    {
        return ['mail', 'database'];
    }

    // Hoe ziet de e-mail eruit?
    public function toMail(object $notifiable): MailMessage
    {
        return (new MailMessage)
                    ->line('Je factuur is betaald.')
                    ->action('Bekijk Factuur', url('/invoices/'.$this->invoice->id))
                    ->line('Bedankt voor het vertrouwen!');
    }

    // Hoe ziet de database melding eruit? (Voor in je dashboard)
    public function toArray(object $notifiable): array
    {
        return [
            'invoice_id' => $this->invoice->id,
            'amount' => $this->invoice->amount,
            'message' => 'Betaling ontvangen',
        ];
    }
}

Versturen doe je direct op het User model (omdat User de Notifiable trait heeft):


$user->notify(new InvoicePaid($invoice));
