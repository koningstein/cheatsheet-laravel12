Gates: Eenvoudige Toegangscontrole

Naast authenticatie (wie ben je?) hebben we ook autorisatie (wat mag je?) nodig. Laravel biedt hiervoor twee systemen: Gates en Policies. Gates zijn bedoeld voor eenvoudige, losstaande checks die niet direct gekoppeld zijn aan een specifiek Model. Denk aan: "Mag deze gebruiker het admin-paneel zien?" of "Mag deze gebruiker bèta-functies gebruiken?".

Stap 1: Een Gate Definiëren

Je definieert Gates in de boot methode van je AppServiceProvider. Een Gate is simpelweg een functie (Closure) die een gebruiker als argument krijgt en true of false teruggeeft. Laravel injecteert de ingelogde gebruiker automatisch.

Bestand: app/Providers/AppServiceProvider.php

namespace App\Providers;

use Illuminate\Support\ServiceProvider;
use Illuminate\Support\Facades\Gate;
use App\Models\User;

class AppServiceProvider extends ServiceProvider
{
    public function boot(): void
    {
        // Definieer een gate genaamd 'view-admin'
        Gate::define('view-admin', function (User $user) {
            // Check logic: bijv. een 'is_admin' kolom in je database
            return $user->is_admin === true;
        });

        // Een gate met argumenten
        Gate::define('edit-settings', function (User $user, $role) {
            return $user->role === $role;
        });
    }
}

Stap 2: Gates Gebruiken in Livewire/PHP

Je kunt overal in je code controleren of een Gate slaagt. De meest gebruikte manier in Controllers of Livewire components is de Gate::authorize methode. Als de check faalt, gooit Laravel direct een 403 Forbidden error en stopt de uitvoering.

Bestand: app/Livewire/Admin/Dashboard.php

use Illuminate\Support\Facades\Gate;

public function mount()
{
    // Optie 1: Harde check (gooit 403 error)
    Gate::authorize('view-admin');

    // Optie 2: Zachte check (voor if/else logica)
    if (Gate::allows('view-admin')) {
        // ...
    }

    if (Gate::denies('view-admin')) {
        abort(403);
    }
}

Stap 3: Gates Gebruiken in Blade

In je views wil je vaak elementen (zoals knoppen of menu-items) verbergen voor gebruikers die geen rechten hebben. Laravel heeft hier de handige @can directive voor. Alles tussen @can en @endcan wordt alleen getoond als de Gate true teruggeeft.

Bestand: resources/views/layouts/navigation.blade.php

<nav>
    <a href="/dashboard">Dashboard</a>

    {{-- Deze link is onzichtbaar voor niet-admins --}}
    @can('view-admin')
        <a href="/admin" class="text-red-600">
            Admin Panel
        </a>
    @endcan
</nav>
