Migrations & Datatypes

Migrations zijn versiebeheer voor je database. Ze zorgen ervoor dat je tabellen kunt aanmaken en wijzigen op een gestructureerde manier.

1. Migration Aanmaken

Je maakt een migratie meestal samen met een model, maar het kan ook los.


# Samen met model (aanbevolen)
php artisan make:model Product -m

# Losse migratie
php artisan make:migration create_products_table

2. Veelgebruikte Kolomtypes

In de up() methode definieer je de kolommen.

Bestand: database/migrations/xxxx_create_products_table.php

public function up(): void
{
    Schema::create('products', function (Blueprint $table) {
        // Primary Key (Auto Increment BigInt)
        $table->id();

        // Tekst
        $table->string('name', 100);       // VARCHAR(100)
        $table->text('description');       // TEXT (veel data)

        // Nummers
        $table->integer('stock');          // INT
        $table->decimal('price', 8, 2);    // DECIMAL(8,2) -> bijv. 123456.78
        $table->boolean('is_active');      // TINYINT(1) (0 of 1)

        // Datums
        $table->date('release_date');      // DATE (2023-01-01)
        $table->dateTime('published_at');  // DATETIME (2023-01-01 12:00:00)

        // Created_at & Updated_at
        $table->timestamps();
    });
}

3. Modifiers (Aanpassingen)

Je kunt eigenschappen aan kolommen toevoegen (chainen).

Bestand: database/migrations/xxxx_create_users_table.php

$table->string('email')->unique();            // Moet uniek zijn
$table->string('phone')->nullable();          // Mag leeg (NULL) zijn
$table->boolean('is_admin')->default(false);  // Standaardwaarde
$table->integer('score')->unsigned();         // Alleen positieve getallen

4. Uitvoeren


php artisan migrate        # Voer uit
php artisan migrate:fresh  # Gooi alles weg en begin opnieuw
