Mass Assignment

Als je Product::create($data) of $product->update($data) gebruikt, heet dat Mass Assignment. Laravel beschermt je standaard tegen het onbedoeld overschrijven van kolommen.

1. $fillable (Allow-list)

Je geeft expliciet aan welke kolommen gewijzigd mogen worden. Dit is de veiligste optie.

Bestand: app/Models/Product.php

protected $fillable = [
    'name',
    'price',
    'description'
];
// 'is_admin' staat er niet bij, dus kan niet via formulier gezet worden!

2. $guarded (Block-list)

Je geeft aan welke kolommen NIET gewijzigd mogen worden. Vaak zetten we dit op een lege array, wat betekent: "Ik let zelf wel op wat ik doorgeef".

Bestand: app/Models/Product.php

// Alles mag ingevuld worden (gevaarlijker, maar makkelijker)
protected $guarded = [];

// Of specifieke beveiliging:
// protected $guarded = ['id', 'is_admin'];
Let op: Als je $guarded = [] gebruikt, moet je ALTIJD $request->validate() of een FormRequest gebruiken om te filteren wat er binnenkomt. Gebruik nooit direct $request->all() zonder validatie!
