M1.1: Model & Migration

Leeruitkomsten

  • Ik kan een model aanmaken met een Artisan commando.
  • Ik kan een migration aanmaken met een Artisan commando.
  • Ik kan kolommen toevoegen aan een migration.
  • Ik kan migrations uitvoeren.

1. Het Concept: MVC & Data

Laravel werkt volgens het MVC-principe. De Model is verantwoordelijk voor alle data-interactie. Elke tabel in je database heeft een eigen Model.

Naming Conventions:
  • Model: Enkelvoud, PascalCase (bijv. Category)
  • Tabel: Meervoud, snake_case (bijv. categories)

2. Aanmaken met Artisan

We maken een Model en Migration tegelijk met de -m vlag. In dit voorbeeld maken we een Category.


php artisan make:model Category -m

3. Migratie Bewerken

In database/migrations/ vind je nu een bestand. Hier definiëren we de kolommen. Standaard is id() en timestamps() aanwezig. We voegen een name toe.


public function up(): void
{
    Schema::create('categories', function (Blueprint $table) {
        $table->id();

        // Voeg hier je eigen kolommen toe
        $table->string('name', 75); // Varchar(75)

        $table->timestamps();
    });
}

4. Uitvoeren

Om de tabel daadwerkelijk in de database te zetten:

php artisan migrate

Heb je een fout gemaakt? Pas het bestand aan en draai:

php artisan migrate:fresh
