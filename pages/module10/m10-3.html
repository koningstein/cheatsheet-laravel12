Form Request Classes

Validatie direct in de controller is prima voor kleine formulieren, maar je controller wordt snel rommelig ("Fat Controller"). De professionele manier in Laravel is het gebruik van Form Requests. Hiermee verplaats je de validatielogica naar een aparte class.

Aanmaken

Door de naam te laten beginnen met het model (Project...) staan straks je Store en Update requests netjes bij elkaar in de map.

Terminal
php artisan make:request ProjectStoreRequest

De Class Configureren

De gegenereerde class heeft twee methodes: authorize() en rules().

Bestand: app/Http/Requests/ProjectStoreRequest.php

namespace App\Http\Requests;

use Illuminate\Foundation\Http\FormRequest;

class ProjectStoreRequest extends FormRequest
{
    // Bepaal of de gebruiker dit mag doen.
    // Zet op 'true' als je permissies elders regelt (of als het openbaar is).
    public function authorize(): bool
    {
        return true;
    }

    // Hier komen je validatieregels
    public function rules(): array
    {
        return [
            'title'       => 'required|min:3|max:255',
            'description' => 'required',
            'budget'      => 'nullable|numeric',
        ];
    }
}

Gebruik in Controller

Je vervangt in je controller de standaard Request door jouw nieuwe ProjectStoreRequest. Laravel voert de validatie automatisch uit voordat de methode begint. Als de validatie faalt, wordt de code in de methode niet eens uitgevoerd.

Bestand: app/Http/Controllers/ProjectController.php

// Vergeet niet te importeren!
use App\Http\Requests\ProjectStoreRequest;

public function store(ProjectStoreRequest $request)
{
    // Als we hier zijn, is de validatie al geslaagd!

    // Haal de gevalideerde data op
    Project::create($request->validated());

    return redirect()->route('projects.index');
}
