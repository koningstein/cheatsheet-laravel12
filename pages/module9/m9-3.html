Stacks & Push

Soms heb je op één specifieke pagina een stukje JavaScript of CSS nodig (bijvoorbeeld voor een grafiek), maar je wilt die zware bestanden niet op elke pagina inladen. Met Stacks kun je content vanuit een sub-pagina "injecteren" naar een specifieke plek in de master layout.

1. De Stack definiëren (Layout)

In je master layout bepaal je waar de extra code terecht moet komen. Meestal is dit in de <head> (voor CSS) of vlak voor </body> (voor JS).

Bestand: resources/views/layouts/app.blade.php

<!DOCTYPE html>
<html>
<head>
    <title>App</title>
    @vite(['resources/css/app.css'])

    {{-- Hier reserveren we plek voor extra styles --}}
    @stack('styles')
</head>
<body>

    @yield('content')

    {{-- Hier reserveren we plek voor extra scripts --}}
    @stack('scripts')

</body>
</html>

2. Pushen naar de Stack (View)

Op de specifieke pagina waar je de grafiekbibliotheek nodig hebt, gebruik je @push.

Bestand: resources/views/dashboard.blade.php

@extends('layouts.app')

@section('content')
    <h1>Dashboard Statistieken</h1>
    <canvas id="myChart"></canvas>
@endsection

{{-- Dit wordt naar de @stack('scripts') in de layout geduwd --}}
@push('scripts')
    <script src="https://cdn.jsdelivr.net/npm/chart.js"></script>
    <script>
        // Initialiseer grafiek
    </script>
@endpush

3. Volgorde (Prepend)

Als de volgorde uitmaakt, kun je ook @prepend gebruiken. Dit zet de code vooraan in de stack in plaats van achteraan. Dit is handig als je een variabele moet definiëren voordat een ander script laadt.
