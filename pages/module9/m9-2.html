Components & Slots

Components zijn de moderne manier om herbruikbare UI-elementen te maken in Laravel (zoals knoppen, alerts, of complete layouts). Ze lijken op HTML-tags.

1. Component Aanmaken

Je kunt een component maken via de terminal. Dit maakt een view en (optioneel) een class.

php artisan make:component Alert --view

De --view vlag zorgt ervoor dat er alleen een Blade-bestand wordt gemaakt (Anonymous Component), wat vaak genoeg is.

2. De Component Structuur ($slot)

De variabele $slot bevat alles wat je tussen de openings- en sluitingstags van je component zet.

Bestand: resources/views/components/alert.blade.php

<div class="bg-red-100 border border-red-400 text-red-700 px-4 py-3 rounded">
    <strong>Let op!</strong>
    <span class="block sm:inline">{{ $slot }}</span>
</div>

3. Gebruik in een View

Je roept een component aan met x- gevolgd door de naam.

Bestand: resources/views/home.blade.php

<x-alert>
    Er is iets fout gegaan bij het opslaan!
</x-alert>

4. Props (Variabelen doorgeven)

Soms wil je data meegeven, zoals het type alert (info, warning, danger). Gebruik @props om standaardwaarden in te stellen.

Bestand: resources/views/components/alert.blade.php

@props(['type' => 'info']) {{-- Standaard is 'info' --}}

@php
    // Bepaal de kleur op basis van het type
    $colors = match($type) {
        'danger' => 'bg-red-100 text-red-700 border-red-400',
        'success' => 'bg-green-100 text-green-700 border-green-400',
        default => 'bg-blue-100 text-blue-700 border-blue-400',
    };
@endphp

<div {{ $attributes->merge(['class' => "border px-4 py-3 rounded $colors"]) }}>
    {{ $slot }}
</div>

Gebruik:


<x-alert type="success" class="mb-4">
    Het is gelukt!
</x-alert>

Let op: $attributes->merge zorgt ervoor dat extra classes (zoals mb-4) netjes worden samengevoegd met de standaard classes.
