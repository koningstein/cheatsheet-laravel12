M2.6: Rolgebaseerde Toegang

We hebben de routes beveiligd, maar soms wil je in de view knoppen verbergen of in de controller specifieke methodes beveiligen.

Stap 1: Blade Directives (@role & @can)

In resources/views/admin/categories/index.blade.php willen we de "Delete" knop verbergen voor gebruikers die dit niet mogen.

Je kunt @role('admin') gebruiken, maar beter is @can('delete category'). Dit kijkt naar de permissie, wat flexibeler is.


<td>
    <a href="...">Edit</a>

    @can('delete category')
        | <a href="...">Delete</a>
    @endcan
</td>

Stap 2: Controller Beveiliging (Middleware)

In de CategoryController moeten we ook checks inbouwen. We gebruiken hiervoor de HasMiddleware interface (nieuw in Laravel 11/12).


use Illuminate\Routing\Controllers\HasMiddleware;
use Illuminate\Routing\Controllers\Middleware;
use Spatie\Permission\Middleware\PermissionMiddleware;

class CategoryController extends Controller implements HasMiddleware
{
    public static function middleware(): array
    {
        return [
            new Middleware(PermissionMiddleware::using('index category'), only:['index']),
            new Middleware(PermissionMiddleware::using('create category'), only:['create', 'store']),
            new Middleware(PermissionMiddleware::using('edit category'), only:['edit', 'update']),
            new Middleware(PermissionMiddleware::using('delete category'), only:['delete', 'destroy']),
        ];
    }

    // ... rest van de controller ...
}
