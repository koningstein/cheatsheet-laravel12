M2.4: Aanmaken van Users

Om te kunnen inloggen en testen, maken we vaste gebruikers aan via een seeder. We koppelen ze direct aan de rollen die we in M2.3 hebben gemaakt.

Stap 1: UserSeeder Aanmaken

php artisan make:seeder UserSeeder

Stap 2: Users Toevoegen

In database/seeders/UserSeeder.php gebruiken we de factory om specifieke users te maken. We overschrijven de naam, email en wachtwoord.


use App\Models\User;
use Illuminate\Support\Facades\Hash;

public function run(): void
{
    // Maak Admin
    $admin = User::factory()->create([
        'name' => 'Admin',
        'email' => 'admin@webshop.nl',
        'password' => Hash::make('test1234')
    ]);
    $admin->assignRole('admin');

    // Maak Sales
    $sales = User::factory()->create([
        'name' => 'Sales',
        'email' => 'sales@webshop.nl',
        'password' => Hash::make('test1234')
    ]);
    $sales->assignRole('sales');

    // Maak Customer
    $customer = User::factory()->create([
        'name' => 'Customer',
        'email' => 'customer@webshop.nl',
        'password' => Hash::make('test1234')
    ]);
    $customer->assignRole('customer');
}

Stap 3: DatabaseSeeder Configureren

Zorg dat de seeders in de juiste volgorde staan in database/seeders/DatabaseSeeder.php. Eerst permissies, dan users!


$this->call([
    RoleAndPermissionSeeder::class,
    UserSeeder::class,
    // andere seeders...
]);

Stap 4: Uitvoeren

php artisan migrate:fresh --seed
