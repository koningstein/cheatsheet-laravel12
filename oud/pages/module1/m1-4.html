M1.4: Resource Controller - Index

Leeruitkomsten

  • Ik kan een resource controller aanmaken.
  • Ik kan data ophalen en doorgeven aan een view.
  • Ik kan een index view maken met een tabel.

Stap 1: Controller Aanmaken

Een Resource Controller bevat standaard al methodes voor alle CRUD-acties. We maken deze in een submap Admin om het netjes te houden.

php artisan make:controller --model=Category Admin/CategoryController

Stap 2: Route Registreren

In routes/web.php registreren we de resource route. Dit maakt in één keer alle routes aan (/admin/categories, /create, etc.).


use App\Http\Controllers\Admin\CategoryController;

Route::resource('/admin/categories', CategoryController::class);

Stap 3: Data Ophalen

In de index() methode halen we alle rijen op uit de database met het Model.


public function index()
{
    // Haal alle categorieën op
    $categories = Category::all();

    // Geef de data door aan de view
    return view('admin.categories.index', [
        'categories' => $categories
    ]);
}

Stap 4: De Index View

Maak resources/views/admin/categories/index.blade.php. We gebruiken @foreach om door de data te loopen.


@extends('layouts.layoutadmin')

@section('content')
    <h1>Overzicht Categorieën</h1>

    <table class="w-full text-left bg-white">
        <thead>
            <tr>
                <th>Naam</th>
                <th>Acties</th>
            </tr>
        </thead>
        <tbody>
            @foreach($categories as $category)
                <tr class="border-b">
                    <td>{{ $category->name }}</td>
                    <td>Edit | Delete</td>
                </tr>
            @endforeach
        </tbody>
    </table>
@endsection
