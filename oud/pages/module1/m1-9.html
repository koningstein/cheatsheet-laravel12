M1.9: Delete & Destroy

Leeruitkomsten

  • Ik kan een delete bevestigingspagina maken.
  • Ik kan de DELETE method gebruiken.
  • Ik kan records verwijderen uit de database.

Stap 1: Extra Route

Omdat we een bevestigingsscherm willen ("Weet u het zeker?"), hebben we een extra route nodig. Plaats deze boven de resource route in web.php.


// Route voor het bevestigingsscherm
Route::get('/admin/categories/{category}/delete',
    [CategoryController::class, 'delete'])
    ->name('categories.delete');

Route::resource('/admin/categories', CategoryController::class);

Stap 2: Delete Methode

Deze methode toont alleen het scherm, hij verwijdert nog niets.


public function delete(Category $category)
{
    return view('admin.categories.delete', [
        'category' => $category
    ]);
}

Stap 3: Delete View

Maak delete.blade.php. Hier gebruik je een formulier met @method('DELETE').


@extends('layouts.layoutadmin')

@section('content')
    <h1 class="text-red-600">Verwijderen?</h1>
    <p>Weet je zeker dat je {{ $category->name }} wilt verwijderen?</p>

    <form action="{{ route('categories.destroy', $category) }}" method="POST">
        @csrf
        @method('DELETE')

        <button type="submit" class="bg-red-500 text-white p-2">
            Ja, weg ermee
        </button>

        <a href="{{ route('categories.index') }}">Annuleren</a>
    </form>
@endsection

Stap 4: Destroy Methode

Dit is de methode die daadwerkelijk verwijdert.


public function destroy(Category $category)
{
    $category->delete();

    return to_route('categories.index')
            ->with('status', 'Categorie verwijderd!');
}
