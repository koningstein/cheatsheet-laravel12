Veilig omgaan met Wachtwoorden

Een van de grootste beveiligingsrisico's in webapplicaties is het lekken van gebruikerswachtwoorden. Als ontwikkelaar heb je de verantwoordelijkheid om wachtwoorden nooit in platte tekst (leesbaar) op te slaan in de database. Als je database ooit gestolen wordt, liggen alle wachtwoorden op straat.

Waarom: Hashing

De oplossing is Hashing. Een hash-functie maakt van een invoer (bijv. "geheim123") een lange, onleesbare reeks tekens (bijv. `$2y$10$92IXUNpkj...`). Het unieke van hashing is dat het eenrichtingsverkeer is. Je kunt van de hash nooit meer terugrekenen naar "geheim123".

Bij het inloggen hasht Laravel het ingevoerde wachtwoord opnieuw en vergelijkt het resultaat met de hash in de database. Als ze matchen, is het wachtwoord juist.

Hoe: De Hash Facade & Validatie

In Laravel 12 heb je krachtige tools om dit veilig te doen. Laten we kijken naar een Livewire component waarmee een gebruiker zijn wachtwoord kan wijzigen.

Bestand: app/Livewire/Settings/UpdatePassword.php

namespace App\Livewire\Settings;

use Livewire\Component;
use Illuminate\Support\Facades\Hash;
use Illuminate\Validation\Rules\Password;

class UpdatePassword extends Component
{
    public string $current_password = '';
    public string $password = '';
    public string $password_confirmation = '';

    public function update()
    {
        // 1. Validatie
        // Laravel heeft een speciale regel 'current_password'.
        // Deze checkt AUTOMATISCH of het veld overeenkomt met het
        // huidige wachtwoord van de ingelogde gebruiker.
        // Voor het nieuwe wachtwoord gebruiken we het Password object voor eisen
        // (minimaal 8 tekens, letters, cijfers, etc.).

        $validated = $this->validate([
            'current_password' => ['required', 'current_password'],
            'password' => ['required', Password::defaults(), 'confirmed'],
        ]);

        // 2. Het wachtwoord updaten
        // We gebruiken Hash::make() om het nieuwe wachtwoord te hashen
        // voordat we het in de database stoppen.

        auth()->user()->update([
            'password' => Hash::make($validated['password']),
        ]);

        // 3. Reset de velden en geef feedback
        $this->reset();
        $this->dispatch('password-updated');
    }
}

Handmatig Hashen en Checken

Hoewel de `current_password` validatieregel veel werk uit handen neemt, moet je soms handmatig met hashes werken. Hiervoor gebruik je de Hash facade.

  • Hash::make('tekst'): Maakt een nieuwe hash (gebruik bij Registratie of Update).
  • Hash::check('tekst', $hash): Controleert of de tekst overeenkomt met de hash (geeft true/false).
Voorbeeld: Handmatige check

if (Hash::check('SuperGeheim', $user->password)) {
    // Het wachtwoord klopt!
}
