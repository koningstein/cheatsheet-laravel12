M5.2: Order Seeds

Leeruitkomsten

  • Ik kan seeders maken voor statische data (States).
  • Ik kan de DatabaseSeeder configureren voor de juiste volgorde.

Stap 1: State Seeder

Voordat we orders kunnen maken, moeten er statussen zijn. Maak een seeder aan:

php artisan make:seeder StateSeeder

Stap 2: Statussen Vullen

We vullen de tabel met vaste waardes.

Bestand: database/seeders/StateSeeder.php

use App\Models\State;

public function run(): void
{
    // Vaste statussen aanmaken
    State::create(['name' => 'Pending']); // ID 1
    State::create(['name' => 'Paid']);    // ID 2
    State::create(['name' => 'Shipped']); // ID 3
    State::create(['name' => 'Delivered']); // ID 4
}

Stap 3: DatabaseSeeder Aanpassen

Zorg dat de StateSeeder wordt aangeroepen in de DatabaseSeeder. De volgorde is belangrijk: States moeten bestaan voordat je Orders (of testdata daarvan) zou seeden.

Bestand: database/seeders/DatabaseSeeder.php

public function run(): void
{
    $this->call([
        RoleAndPermissionSeeder::class,
        UserSeeder::class,
        ProductSeeder::class, // Maakt categorieën, producten en prijzen
        StateSeeder::class,   // NIEUW: Maakt statussen
    ]);
}

Stap 4: Uitvoeren

Ververs de database om de nieuwe tabellen en data te krijgen.

php artisan migrate:fresh --seed
