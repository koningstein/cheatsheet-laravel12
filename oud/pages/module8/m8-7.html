Dependency Injection

Laravel's Service Container is krachtig. Je hoeft klassen bijna nooit zelf te instantiëren (met new). Je "vraagt" er gewoon om in je controller, en Laravel "injecteert" ze.

1. Method Injection

Dit is de meest voorkomende vorm, bijvoorbeeld voor de Request klasse. Je typt de classnaam voor de variabele in de methode.

Bestand: app/Http/Controllers/UserController.php

use Illuminate\Http\Request;

// Laravel ziet 'Request' en geeft automatisch het huidige request object mee
public function store(Request $request)
{
    $name = $request->input('name');
}

2. Constructor Injection

Als je een service of repository in meerdere methodes van je controller nodig hebt, injecteer je hem in de constructor.

Bestand: app/Http/Controllers/OrderController.php

use App\Services\PaymentGateway;

class OrderController extends Controller
{
    // PHP 8+ Constructor Property Promotion
    // Laravel injecteert automatisch een instantie van PaymentGateway
    public function __construct(
        private PaymentGateway $payment
    ) {}

    public function store()
    {
        // Je kunt de service nu overal gebruiken
        $this->payment->charge(100);
    }
}
