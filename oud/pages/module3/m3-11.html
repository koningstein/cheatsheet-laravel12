M3.11: Delete View met Relaties

Leeruitkomsten

  • Ik kan een delete view maken die gerelateerde informatie toont.
  • Ik kan disabled fields gebruiken voor dropdowns en inputs.

Stap 1: Controller (Delete)

We halen het product op om te tonen wat er verwijderd gaat worden.


public function delete(Product $product)
{
    return view('admin.products.delete', ['product' => $product]);
}

Stap 2: View: Disabled Fields

Het formulier lijkt op edit, maar alles is disabled. Bij de categorie hoeven we geen dropdown te tonen, we kunnen simpelweg de naam van de huidige categorie tonen.


<form action="{{ route('products.destroy', $product) }}" method="POST">
    @csrf @method('DELETE')

    <label>Naam:</label>
    <input type="text" value="{{ $product->name }}" disabled>

    <label>Categorie:</label>
    <!-- Gewoon de naam tonen, geen select nodig -->
    <input type="text" value="{{ $product->category->name }}" disabled>

    <label>Prijs:</label>
    <input type="text" value="{{ $product->latest_price->price }}" disabled>

    <button type="submit" class="bg-red-500">Verwijderen</button>
</form>
Opdracht 26: Maak de Delete view voor Tasks. Zorg dat alle velden (inclusief Project en Activity) disabled zijn en de huidige waarde tonen.
