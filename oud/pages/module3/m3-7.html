M3.7: Store & Validatie

Leeruitkomsten

  • Ik kan een StoreRequest maken met complexe validatie (foreign keys).
  • Ik kan data opslaan in meerdere tabellen (Product + Price).
  • Ik kan transacties of volgordelijkheid beheren.

Stap 1: Validatie Request

We maken ProductStoreRequest. Let op de validatie van de foreign key: deze MOET bestaan in de database (exists).


public function rules(): array
{
    return [
        'name' => 'required|string|max:75',
        // Foreign key validatie: moet bestaan in tabel categories, kolom id
        'category_id' => 'required|integer|exists:categories,id',
        // Prijs validatie (voor de andere tabel)
        'price' => 'required|numeric|min:0|max:999999.99',
    ];
}

Stap 2: Opslaan (Store Methode)

In de controller moeten we twee dingen doen: eerst het Product aanmaken, en daarna de Prijs (want de prijs heeft het product_id nodig).


public function store(ProductStoreRequest $request)
{
    // 1. Maak Product
    $product = new Product();
    $product->name = $request->name;
    $product->category_id = $request->category_id;
    $product->save(); // Nu heeft $product een ID!

    // 2. Maak Prijs (gekoppeld aan product)
    $price = new Price();
    $price->price = $request->price;
    $price->effdate = now(); // Huidige datum/tijd
    $price->product_id = $product->id; // Koppel ID van stap 1
    $price->save();

    return to_route('products.index')
            ->with('status', 'Product en prijs opgeslagen!');
}
Opdracht 22: Pas dit toe op Tasks. Een Task heeft foreign keys naar project_id en activity_id (verplicht) en user_id (optioneel/nullable). Vergeet de exists validatie niet!
