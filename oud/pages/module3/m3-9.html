M3.9: Edit & Pre-select

Leeruitkomsten

  • Ik kan een edit formulier maken met dropdowns voor relaties.
  • Ik kan de huidige foreign key waarde pre-selecteren.
  • Ik kan oude input (old()) combineren met database waarden.

Stap 1: Controller (Edit)

Voor de edit pagina hebben we het product nodig (voor de values) én de lijst met categorieën (voor de dropdown).


public function edit(Product $product)
{
    $categories = Category::all();

    return view('admin.products.edit', [
        'product' => $product,
        'categories' => $categories
    ]);
}

Stap 2: View: Pre-select in Dropdown

Dit is de truc: we moeten controleren of de optie in de loop gelijk is aan de category_id van het product. We gebruiken old() met als tweede parameter de database waarde.


<label>Categorie:</label>
<select name="category_id" class="border p-2">

    @foreach($categories as $category)
        <option value="{{ $category->id }}"
            {{-- Check: is dit de oude input? Of is dit de waarde uit de DB? --}}
            @selected(old('category_id', $product->category_id) == $category->id)
        >
            {{ $category->name }}
        </option>
    @endforeach

</select>

Stap 3: Prijs veld

Voor de prijs halen we de laatste prijs op via de relatie latest_price.


<input type="number" name="price"
       value="{{ old('price', $product->latest_price->price) }}">
