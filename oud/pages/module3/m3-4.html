M3.4: Index & Eager Loading

Leeruitkomsten

  • Ik kan gerelateerde data (categorie, prijs) tonen in een tabel.
  • Ik begrijp het N+1 probleem.
  • Ik kan Eager Loading (with()) gebruiken om dit op te lossen.

1. Het N+1 Probleem

Als we in de view voor elk product de categorie naam opvragen ($product->category->name), voert Laravel op de achtergrond steeds een nieuwe query uit. Bij 100 producten zijn dat 101 queries! Dit maakt de applicatie traag.

2. Eager Loading

Met with() halen we de relaties in één keer op. Standaard is net zoals database relaties een 1 op 1 of 1 op veel relate. Bij 1 op veel moeten we aan de ene kant HasMany gebruiken, en de andere kan BelongsTo. We maken ook meteen een speciale relatie latest_price om de huidige prijs te tonen.


// app/Models/Product.php
public function category(): BelongsTo
{
    return $this->BelongsTo(Category::class);
}
        

// app/Models/Product.php
public function latest_price(): HasOne
{
    return $this->hasOne(Price::class)->orderBy('effdate', 'desc');
}

In de controller:


public function index()
{
    // Haal alles in 1x op
    $products = Product::with('category', 'latest_price')->paginate(15);

    return view('admin.products.index', ['products' => $products]);
}
