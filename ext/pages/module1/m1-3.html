Laravel Lay-out & Tailwind

Voordat we beginnen met programmeren, zetten we een goede basis lay-out op. In Laravel gebruiken we Blade templates en een Masterpage concept om herhaling te voorkomen (DRY principe). We maken gebruik van de Livewire starterkit voor de basis authenticatie en styling.

Livewire Starterkit

We gebruiken de Livewire starterkit (vervanger voor Breeze in Laravel 12). Dit pakket geeft ons direct:

  • Authenticatie functionaliteit (Login, Register)
  • Tailwind CSS configuratie
  • Vite bundler (voor assets)

De bestanden vind je in app/Http/Controllers/Auth en de views in resources/views/livewire/auth.

Template Splitting

We splitsen onze layout op in kleine, herbruikbare bestanden. Dit implementeert het Composite Pattern. Zo blijft je code overzichtelijk en onderhoudbaar.

Maak de volgende structuur in resources/views/layouts/:

  • layoutadmin.blade.php (De masterpage)
  • base/start.blade.php (HTML head & styles)
  • base/navbar.blade.php (Bovenbalk)
  • base/sidebar.blade.php (Zijmenu)
  • base/end.blade.php (Scripts & footer)

De Masterpage

De masterpage (layoutadmin.blade.php) knoopt alles aan elkaar. We gebruiken @include voor de vaste onderdelen en @yield voor de dynamische content van subpagina's.


@include('layouts.base.start')
@include('layouts.base.navbar')

<div class="h-screen flex flex-row flex-wrap">

    @include('layouts.base.sidebar')

    <div class="bg-gray-100 flex-1 p-6 md:mt-16">
        @yield('topmenu')

        @yield('content')
    </div>
    </div>
@include('layouts.base.end')

De Base Bestanden

In start.blade.php laden we de assets via Vite. De asset() functie zorgt dat links blijven werken, ook in submappen.


<head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Laravel Opdrachten</title>

    @vite(['resources/css/app.css', 'resources/js/app.js'])
</head>
<body class="bg-gray-100">

In navbar.blade.php maken we de logout functionaliteit. Dit moet via een POST request (formulier) voor veiligheid, maar we stylen het als een linkje.


<a href="{{ route('logout') }}"
   onclick="event.preventDefault(); document.getElementById('logout-form').submit();"
   class="...">
   Log out
</a>

<form id="logout-form" action="{{ route('logout') }}" method="POST" style="display: none;">
    @csrf
</form>

In sidebar.blade.php gebruiken we @guest en @else directives om links te tonen op basis van inlogstatus.


@guest
    <!-- Links voor gasten -->
    <p>Guest</p>
    <a href="{{ route('login') }}">Login</a>
@else
    <!-- Links voor ingelogde users (admins) -->
    <p>Admin</p>
    <a href="">Category admin</a>
    <a href="">Product admin</a>
@endguest

Testen & Routes

Om de nieuwe layout te testen, maken we een tijdelijke route aan in routes/web.php. We laden direct de view om te zien of de structuur werkt.


// routes/web.php

Route::get('/admin', function () {
    // We laden direct de layout file om te testen
    return view('layouts.layoutadmin');
})->name('admin');

Start je server en ga naar /admin om het resultaat te bekijken. Je zou nu de sidebar en navbar moeten zien, maar nog zonder content in het midden.
